# Claude Code プロジェクト設定

## 作業再開時（最初に読む）
作業を再開するときは、まず `.claude/PROGRESS.md` を読んで現在地・次のタスク・要対応事項を把握すること。

## プロジェクト要件定義
不明点が生じたら必ず `.claude/project-brief.md` を参照すること。
ブランド・デザイン・機能・技術スタックの決定事項がすべて記載されている。

## コーディング作業のフェーズ管理（必須）

**すべてのコーディング作業**において `~/.claude/skills/coding-phases.md` を適用すること。

- 作業を「設計→実装→スタイリング→動作確認→仕上げ」のフェーズに分ける
- **各フェーズの終わりにユーザーの確認を取ってから次のフェーズへ進む**
- 小さな1行修正・typo修正はフェーズ分け不要

## スキルの自動参照

UI・フロントエンドのコードを構築・修正する際は、必ず以下のスキルファイルを参照してください：

- `~/.claude/skills/frontend-design.md` — デザインルール・コーディング規約
- `~/.claude/skills/coding-phases.md` — コーディング作業のフェーズ管理

### 自動参照のトリガー
以下の作業を行うときは`frontend-design`スキルを適用すること：
- コンポーネントの新規作成・修正
- ページレイアウトの構築
- スタイリングの追加・変更
- UIのレスポンシブ対応
- アクセシビリティの実装
